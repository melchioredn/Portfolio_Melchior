Déploiement et Configuration d’Apache2 sous Ubuntu

1. Préparation de l’environnement

Avant de commencer, j’ai utilisé un système Ubuntu 22.04 LTS à jour. J’ai ouvert le terminal et vérifié que les paquets essentiels étaient installés :

  • sudo apt update → mise à jour des dépôts
  • sudo apt upgrade → mise à jour des paquets
Ensuite, j’ai planifié la configuration du serveur web Apache2.

2. Installation d’Apache2

Pour installer Apache2, j’ai utilisé la commande : 

sudo apt install apache2 -y
Après installation, j’ai vérifié que le service était actif : 
sudo systemctl status apache2

3. Configuration des fichiers de base

J’ai configuré le serveur en éditant les fichiers principaux :

  • /etc/apache2/apache2.conf → configuration globale
  • /etc/apache2/ports.conf → définition du port (80 par défaut)
  • /var/www/html → emplacement des fichiers web
Pour tester, j’ai créé un fichier index.html dans /var/www/html : 
echo "<h1>Bienvenue sur Apache2</h1>" | sudo tee /var/www/html/index.html

4. Gestion des modules Apache2

Certains modules sont essentiels pour le fonctionnement des sites web :

  • Activer le module rewrite : 
    sudo a2enmod rewrite
  • Redémarrer Apache2 après activation : 
    sudo systemctl restart apache2

5. Configuration des Virtual Hosts

J’ai créé un Virtual Host pour un site exemple : 

sudo nano /etc/apache2/sites-available/mon-site.conf
Contenu du fichier : 
<VirtualHost *:80>
      ServerAdmin admin@example.com
      ServerName mon-site.local
      DocumentRoot /var/www/mon-site
      ErrorLog ${APACHE_LOG_DIR}/error.log
      CustomLog ${APACHE_LOG_DIR}/access.log combined
  </VirtualHost>
Puis activation du site et rechargement d’Apache2 : 
sudo a2ensite mon-site.conf
  sudo systemctl reload apache2

6. Sécurisation et pare-feu

J’ai ouvert le port 80 et 443 dans UFW : 

sudo ufw allow 'Apache Full'
Puis vérification de l’état du pare-feu : 
sudo ufw status

7. Tests et vérifications

Depuis un navigateur sur la même machine ou sur un poste client, j’ai saisi : 👉 http://localhost ou http://mon-site.local après modification du fichier /etc/hosts.

  • Page test affichée correctement → installation réussie
  • Logs Apache consultés : 
    sudo tail -f /var/log/apache2/access.log

8. Conclusion

Le serveur Apache2 est désormais installé et configuré. Je peux héberger plusieurs sites avec Virtual Hosts, activer des modules supplémentaires et sécuriser le serveur via UFW. Ce déploiement m’a permis de comprendre l’architecture d’un serveur web Apache et d’acquérir une expérience pratique dans la gestion des services web sous Linux.

E. R. Melchior EDOUN
