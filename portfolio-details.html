Déploiement et Configuration de pfSense sous VirtualBox

1. Préparation de l’environnement

Avant de commencer, j’ai installé Oracle VM VirtualBox sur mon poste de travail. Cet hyperviseur me permet de Ensuite, j’ai téléchargé l’ISO de pfSense depuis le site officiel (https://www.pfsense.org). J’ai choisi la version AMD64 (architecture 64 bits) avec installateur en image ISO.

2. Création de la machine virtuelle

Dans VirtualBox, j’ai créé une nouvelle VM avec les paramètres suivants :

  •    Nom : pfSense
  •    Type : BSD
  •    Version : FreeBSD (64-bit)
  •    Mémoire RAM : 2048 Mo (2 Go)
  •    Disque dur : 20 Go en format VDI, stockage dynamique
  •    Processeurs : 2 vCPU pour de meilleures performances
Ensuite, j’ai attaché l’ISO de pfSense comme lecteur CD/DVD virtuel.

3. Configuration réseau dans VirtualBox

pfSense a besoin d’au moins deux cartes réseau :

  •    WAN (Internet) → reliée à la carte réseau physique de mon PC (mode pont / bridged adapter)
  •    LAN (Réseau local) → reliée à un réseau interne VirtualBox (Internal Network)
Ainsi, j’ai configuré dans VirtualBox :
  •    Carte réseau 1 → Mode Bridged Adapter (WAN)
  •    Carte réseau 2 → Mode Internal Network (LAN)

4. Installation de pfSense

J’ai démarré la VM et l’installateur pfSense s’est lancé. Étapes effectuées :

  1.  Sélection de Install pfSense
  2.  Choix du clavier (French / FR)
  3.  Partitionnement : Auto (UFS)
  4.  Lancement de l’installation → Copie des fichiers sur le disque
  5.  Redémarrage de la VM

5. Configuration initiale (console pfSense)

Après le redémarrage, j’ai eu accès au menu console de pfSense. J’ai suivi ces étapes :

  • Vérification que l’interface WAN correspond bien à la carte en mode pont
  • Vérification que l’interface LAN correspond à la carte en réseau interne
  • Attribution automatique par pfSense :
    • WAN : IP reçue de ma box internet (via DHCP)
    • LAN : IP statique 192.168.1.1/24

6. Accès à l’interface Web

Sur mon poste hôte, j’ai créé une seconde VM cliente (Windows/Linux) connectée au même réseau interne VirtualBox (Internal Network).
Ensuite, depuis cette VM cliente, j’ai ouvert un navigateur et saisi :
👉 https://192.168.1.1
Un certificat auto-signé m’a été présenté, j’ai accepté l’exception.
Login par défaut :

  • Username : admin
  • Mot de passe : pfsense

7. Configuration via l’assistant (Setup Wizard)

Une fois connecté, j’ai suivi l’assistant de configuration :

  1. Définition du nom d’hôte et du domaine
  2. Configuration du DNS (Google DNS : 8.8.8.8 / 1.1.1.1 de Cloudflare)
  3. Vérification des interfaces WAN et LAN
  4. Modification du mot de passe administrateur

8. Tests et Vérifications

  • Depuis la VM cliente sur le LAN, j’ai testé la connexion Internet → succès
  • J’ai configuré le DHCP Server sur pfSense pour le LAN afin d’attribuer automatiquement des adresses IP aux clients
  • J’ai testé le pare-feu en ajoutant quelques règles simples (ex : bloquer ICMP/ping)

9. Conclusion

J’ai ainsi réussi à déployer pfSense dans un environnement virtualisé avec VirtualBox.
Ce lab me permet de pratiquer :

  • Le pare-feu et le filtrage de paquets
  • La gestion du NAT
  • La configuration du DHCP/DNS interne
  • Les futures fonctionnalités comme le VPN, le portail captif ou la haute disponibilité

E. R. Melchior EDOUN

Analyste Cybersécurité

L’installation de pfSense m’a permis de mettre en place une solution de sécurité réseau fiable et flexible. Grâce à ses fonctions de pare-feu, de VPN et de gestion centralisée (DHCP, NAT, VLAN), j’ai pu sécuriser et organiser efficacement le trafic. En plus d’être une solution open-source économique, pfSense offre des modules complémentaires comme Snort ou pfBlockerNG pour renforcer la protection. Ce déploiement m’a également permis d’acquérir une expérience pratique dans l’administration et la sécurisation d’un réseau professionnel.
