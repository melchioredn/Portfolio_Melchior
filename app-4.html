Déploiement et Configuration de Nginx sous Ubuntu

1. Préparation de l’environnement

J’ai utilisé un système Ubuntu 22.04 LTS à jour. J’ai commencé par mettre à jour les dépôts et les paquets essentiels :

  • sudo apt update → mise à jour des dépôts
  • sudo apt upgrade → mise à jour des paquets
L’environnement était prêt pour installer le serveur web Nginx.

2. Installation de Nginx

Pour installer Nginx, j’ai utilisé : 

sudo apt install nginx -y
Ensuite, j’ai vérifié que le service était actif : 
sudo systemctl status nginx

3. Configuration des fichiers principaux

J’ai configuré Nginx en éditant les fichiers essentiels :

  • /etc/nginx/nginx.conf → configuration globale
  • /etc/nginx/sites-available/default → site par défaut
  • /var/www/html → emplacement des fichiers web
Pour tester, j’ai créé un fichier index.html : 
echo "<h1>Bienvenue sur Nginx</h1>" | sudo tee /var/www/html/index.html

4. Gestion des modules et configurations supplémentaires

Pour activer certaines fonctionnalités et configurations, j’ai modifié :

  • Les directives server et location dans les fichiers de site
  • Redémarrage de Nginx après modification : 
    sudo systemctl restart nginx

5. Configuration des Virtual Hosts (Server Blocks)

J’ai créé un Server Block pour un site exemple : 

sudo nano /etc/nginx/sites-available/mon-site
Contenu du fichier : 
server {
      listen 80;
      server_name mon-site.local;
      root /var/www/mon-site;

      index index.html;

      access_log /var/log/nginx/mon-site.access.log;
      error_log /var/log/nginx/mon-site.error.log;

      location / {
          try_files $uri $uri/ =404;
      }
  }
Activation du site et rechargement de Nginx : 
sudo ln -s /etc/nginx/sites-available/mon-site /etc/nginx/sites-enabled/
  sudo nginx -t
  sudo systemctl reload nginx

6. Sécurisation et pare-feu

J’ai ouvert le port 80 et 443 dans UFW : 

sudo ufw allow 'Nginx Full'
Vérification de l’état du pare-feu : 
sudo ufw status

7. Tests et vérifications

Depuis un navigateur : 👉 http://localhost ou http://mon-site.local après modification du fichier /etc/hosts.

  • Page test affichée correctement → installation réussie
  • Consultation des logs Nginx : 
    sudo tail -f /var/log/nginx/access.log

8. Conclusion

Le serveur Nginx est maintenant installé et configuré. Je peux héberger plusieurs sites via Server Blocks, activer des configurations spécifiques et sécuriser le serveur avec UFW. Ce déploiement m’a permis de comprendre l’architecture d’un serveur Nginx et d’acquérir une expérience pratique dans la gestion des services web sous Linux.

E. R. Melchior EDOUN
