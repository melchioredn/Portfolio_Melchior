Déploiement et Configuration de Centreon sous VirtualBox

1. Préparation de l’environnement

J’ai installé Oracle VM VirtualBox sur mon poste de travail afin de créer un environnement virtualisé. Ensuite, j’ai téléchargé l’ISO de Centreon avec sa distribution Linux (CentOS 7 ou AlmaLinux 8 recommandé). L’objectif est de disposer d’un serveur de supervision complet capable de surveiller les services, équipements réseau et applications.

2. Création de la machine virtuelle

Dans VirtualBox, j’ai créé une VM avec les paramètres suivants :

  •    Nom : Centreon
  •    Type : Linux
  •    Version : Red Hat (64-bit)
  •    Mémoire RAM : 4096 Mo (4 Go)
  •    Disque dur : 40 Go en format VDI, stockage dynamique
  •    Processeurs : 2 vCPU pour de bonnes performances
J’ai attaché l’ISO Linux pour l’installation du serveur de base.

3. Configuration réseau dans VirtualBox

Centreon nécessite une connexion réseau pour superviser les hôtes. J’ai configuré :

  • Carte réseau 1 (LAN) → Mode Bridged Adapter pour accéder au réseau interne et à Internet

4. Installation du système d’exploitation

Après avoir démarré la VM, j’ai installé le système Linux choisi avec ces étapes :

  1. Sélection de la langue et du clavier (French / FR)
  2. Configuration du disque : partitionnement automatique
  3. Installation minimale avec SSH activé
  4. Redémarrage de la VM après installation

5. Installation de Centreon

J’ai suivi les étapes officielles de Centreon :

  1. Ajout des dépôts Centreon
  2. Installation du moteur Centreon et de l’interface Web
  3. Configuration de la base de données MySQL/MariaDB pour stocker les métriques
  4. Installation des modules et plugins de supervision (ex : SNMP, Nagios plugins)

6. Accès à l’interface Web

Depuis un navigateur sur mon poste hôte ou VM cliente, j’ai saisi :
👉 http://IP_Centreon/centreon
Les identifiants par défaut :

  • Username : admin
  • Mot de passe : centreon

7. Configuration via l’assistant (Setup Wizard)

Une fois connecté, j’ai configuré :

  1. Nom du serveur et domaine
  2. Connexion à la base de données Centreon
  3. Configuration des notifications et SMTP
  4. Ajout de l’interface de supervision et des premiers hôtes

8. Ajout et supervision des hôtes

J’ai commencé à superviser des équipements réseau et serveurs :

  • Ajout d’un hôte Windows avec plugin NSClient++
  • Ajout d’un hôte Linux avec NRPE pour surveiller CPU, RAM, disque
  • Création de services et alertes (ex : ping, HTTP, SSH, base de données)

9. Tests et Vérifications

  • Vérification de la réception des métriques sur l’interface Web
  • Test des notifications par email
  • Simulation d’incidents pour vérifier les alertes

10. Conclusion

Le déploiement de Centreon dans un environnement virtualisé m’a permis de créer une solution complète de supervision. Je peux désormais monitorer l’état des services, générer des rapports, recevoir des alertes et optimiser la gestion du réseau. Ce lab est idéal pour pratiquer la configuration de supervision, le monitoring d’hôtes et la mise en place de plugins avancés.

E. R. Melchior EDOUN
