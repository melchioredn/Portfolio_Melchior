Déploiement et Configuration de Bind9 sous VirtualBox

1. Préparation de l’environnement

J’ai installé Oracle VM VirtualBox sur mon poste pour virtualiser un serveur Linux (Debian/Ubuntu). Ensuite, j’ai téléchargé l’ISO Debian depuis le site officiel (https://www.debian.org).

2. Création de la machine virtuelle

Dans VirtualBox, j’ai créé une VM avec ces paramètres :

  •    Nom : DNS-Server
  •    Type : Linux
  •    Version : Debian (64-bit)
  •    Mémoire RAM : 1024 Mo (1 Go)
  •    Disque dur : 10 Go en VDI, stockage dynamique
  •    Processeurs : 1 vCPU suffisant pour un serveur DNS de test
J’ai attaché l’ISO Debian comme lecteur CD/DVD virtuel pour installer le système.

3. Configuration réseau dans VirtualBox

J’ai configuré une carte réseau en Bridged Adapter pour le WAN et une en Internal Network pour le LAN afin de tester la résolution DNS localement.

4. Installation de Debian

Après le démarrage de la VM, j’ai suivi l’installation standard :

  1.  Sélection de la langue et du clavier
  2.  Configuration réseau : IP statique 192.168.1.10 pour le serveur DNS
  3.  Partitionnement : Guidé – tout sur un disque
  4.  Création d’un utilisateur et mot de passe root
  5.  Installation du système minimal

5. Installation de Bind9

Une fois Debian installé, j’ai exécuté :

  • sudo apt update
  • sudo apt install bind9 bind9utils bind9-doc dnsutils -y

6. Configuration des zones DNS

J’ai configuré les fichiers principaux :

  • /etc/bind/named.conf.local → Définition des zones directes et inverses
  • /etc/bind/db.example.com → Zone directe
  • /etc/bind/db.192 → Zone inverse
Exemple d’entrée dans db.example.com : 
        @   IN  SOA ns1.example.com. admin.example.com. (
                2025082001 ; Serial
                3600       ; Refresh
                1800       ; Retry
                604800     ; Expire
                86400 )    ; Minimum
        ;
        @       IN  NS      ns1.example.com.
        ns1     IN  A       192.168.1.10
        www     IN  A       192.168.1.20

7. Vérification et tests

Après avoir redémarré Bind9 :

  • sudo systemctl restart bind9
  • sudo systemctl status bind9
J’ai testé la résolution DNS depuis la VM cliente :
  • nslookup www.example.com 192.168.1.10 → Résultat attendu : 192.168.1.20

8. Conclusion

Ce déploiement m’a permis de mettre en place un serveur DNS fonctionnel avec Bind9 :

  • Résolution directe et inverse
  • Gestion centralisée des noms de domaine internes
  • Expérience pratique sur la configuration et la sécurité de Bind9
